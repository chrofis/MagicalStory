## GEO (Generative Engine Optimization) Strategy

### What is GEO?

Optimizing content so AI assistants (ChatGPT, Google AI Overviews, Perplexity, Claude) reference and recommend MagicalStory.ch when users ask about personalized children's books.

**Why it matters:** Increasingly, parents will ask AI assistants "What's the best personalized children's book for my kid?" instead of Googling it. We need to be in those answers.

### GEO Tactics

**1. Structured Data Excellence**
Every page type gets appropriate schema:
- FAQ schema on all pages with Q&A (theme, comparison, occasion, FAQ)
- Product schema on pricing page and theme pages
- Organization schema on about page
- HowTo schema on "how it works" section
- BreadcrumbList schema on all pages
- Review/AggregateRating schema (when real reviews exist)
- LocalBusiness schema for Swiss presence

**2. Entity Optimization**
- Consistent brand name everywhere: "MagicalStory" (not variations)
- Associate with key entities: "personalized children's books", "AI storybooks", "Swiss children's books"
- Create comprehensive about page (Wikipedia-style: factual, specific, cited)
- Get brand mentioned on relevant Wikipedia list pages if possible
- Ensure consistent NAP (Name, Address, Phone) across all directories

**3. Answer-Optimized Content**
- Structure content as clear Q&A where possible
- Start paragraphs with direct answers, then expand
- Use "MagicalStory is..." and "MagicalStory offers..." phrasing
- Include specific statistics and facts (AI systems prefer citable data)
- Example: "MagicalStory offers 170+ story themes in 3 languages" (specific, citable)

**4. Comparison Content for AI Citation**
- Listicles with clear feature comparisons are AI catnip
- "Best personalized children's books in Switzerland" — structured, factual
- Include MagicalStory with specific, honest differentiators
- AI systems love tables with clear data points

**5. Topical Authority Clusters**
Build interconnected content hubs:

```
"Personalisierte Kinderbücher" (hub page)
├── Theme pages (170+ spokes)
├── Town pages (50+ spokes)
├── Occasion pages (12+ spokes)
├── Comparison pages (8+ spokes)
└── Blog posts (12+ spokes)

"KI-generierte Kinderbücher" (hub page)
├── How it works (spoke)
├── Art styles explained (spoke)
├── Safety & privacy (spoke)
├── Comparison pages (spokes)
└── Behind the scenes (spoke)
```

**6. AI-Friendly Content Patterns**
- Use definition patterns: "MagicalStory is an AI-powered platform that..."
- Include "Founded in Switzerland" for geographic authority
- List specific capabilities with numbers: "170+ themes", "8 art styles", "3 languages"
- Comparison tables with clear winners per category
- FAQ sections that match common AI queries

## Technical SEO Recommendations

### Priority 1: Schema Implementation

```json
// FAQ Schema (add to theme, comparison, occasion, FAQ pages)
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "Ist die erste Geschichte wirklich kostenlos?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Ja, deine erste Geschichte wird in voller Qualität generiert..."
    }
  }]
}

// Product Schema (add to pricing and theme pages)
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Personalisiertes Kinderbuch",
  "description": "KI-illustriertes Kinderbuch mit deinem Kind als Held",
  "brand": {"@type": "Brand", "name": "MagicalStory"},
  "offers": {
    "@type": "AggregateOffer",
    "lowPrice": "0",
    "highPrice": "96",
    "priceCurrency": "CHF"
  }
}

// Organization Schema (about page)
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "MagicalStory",
  "url": "https://magicalstory.ch",
  "description": "AI-powered personalized children's storybooks",
  "foundingLocation": "Switzerland"
}

// BreadcrumbList (all pages)
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [...]
}
```

- Preconnect to Google Fonts, CDN origins
- Target: LCP <2.5s, INP <200ms, CLS <0.1

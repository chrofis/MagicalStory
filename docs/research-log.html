2. FLUX (Runware) — character replacement

Tested: February 2025. Verdict: ❌ rejected for character replacement; ✅ kept for cheap iteration only.

FLUX Kontext via Runware claimed reference-image conditioning for character preservation. In practice referenceImages were silently ignored by the model — the output didn't track the reference identity. Stitching the reference into the prompt as a base64 image worked partially but produced visible seams and ghosting. Not viable for production page generation.

FLUX Dev / Schnell tested for page generation: style and character consistency degraded across pages — same prompt, same seed produced visibly different characters. Storybooks need cross-page consistency; FLUX falls short.

3. Grok Imagine — edit + multi-character

Tested: March 2026. Verdict: ✅ kept for repair / edit; primary for character repair.

Findings:

  • Grok edit endpoint does full regeneration, not true inpainting. The cutout approach (extract bbox + 20% padding, send cutout + avatar, composite back) is what makes it work for repair.
  • Sweet spot: 3-5 characters per generation. Beyond that, identity degrades.
  • Red-box swap test confirmed Grok can replace a character cleanly when the cutout is properly sized.
  • Aurora model is autoregressive — prompt order matters (scene → style → lighting → camera).
  • No negative prompt support — use positive phrasing only.
  • Best for: anime / Studio Ghibli / cinematic / photorealistic. Weak for: flat illustration, watercolor, minimalist (adds unwanted polish).

Grok inpaint can handle structural changes (body rotation, pose, gaze) when instructions use visual identifiers, not character names. Don't route facing-direction issues to a full iterate when an inpaint pass works.

4. Seedream 4.5 (ByteDance)

Tested: February 2026. Verdict: ❌ not integrated.

ByteDance Seedream 4.5 advertises strong character consistency. Pricing and reference-image limits at the time of evaluation didn't beat the current Gemini + Grok mix. Worth re-evaluating if pricing changes or if their character-consistency claims hold up on a real 12-page test.

5. Face embeddings (InsightFace / ArcFace)

Verdict: ❌ not adopted.

Embedding-based identity preservation (compute a face embedding from the user photo, condition generation on that embedding) was investigated as an alternative to the avatar-as-reference approach. Adds significant complexity (model hosting, dimensional alignment per backend) for a quality lift that the current avatar pipeline already achieves at acceptable consistency levels.

7. Fairy tales as a story category

Verdict: ✅ research canonical — Märchen is the next planned story category.

Curated catalog of Swiss + international fairy tales suitable for the platform: title, source culture, age range, key motifs, typical visual elements. Used to seed prompt templates and curated historical_locations for fairy-tale settings (forest, castle, cottage). Wilhelm Tell was the first historical theme; fairy tales are next.

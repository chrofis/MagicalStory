API Reference

REST endpoints exposed by the Express backend, grouped by domain. Mount points live in server.js; per-domain routers live under server/routes/.

Auth: most endpoints require a JWT (Authorization: Bearer <token>) issued by /api/auth/login. Public endpoints are noted explicitly. Routes under /api bypass the staging Basic-Auth gate so XHR calls work without credentials reuse.

Files: server/routes/auth.js, server/routes/user.js.

MethodPathNotes

GET / POST/api/user/shipping-addressRead / update shipping address.

2. Characters & avatars

Files: server/routes/characters.js, server/routes/avatars.js, server/routes/photos.js.

MethodPathNotes
GET/api/charactersList user's characters (metadata only).
POST/api/charactersCreate character (name, traits, relationships).
GET/api/characters/:idFull character incl. avatars (R2 URLs).
PATCH/api/characters/:idEdit traits/clothing/relationships.
DELETE/api/characters/:idSoft delete.
POST/api/analyze-photoSend photo bytes → Python analyzer → traits + face.
POST/api/avatar-promptBuild avatar prompt from traits.
POST/api/generate-clothing-avatarsSpawn body+face crops per clothing slot.
POST/api/avatars/:characterIdGenerate / regenerate the standard 2×2 grid avatar.

See also: character system for the data model and avatar lifecycle.

MethodPathNotes

POST/:id/repair-workflow/character-repairGrok cutout / blended / blackout repair.

POST/:id/covers/:coverType/active-imageSwitch active cover version.

See also: image quality stack for the scoring + repair logic that drives these endpoints.

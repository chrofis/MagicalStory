6. Trial flow

File: server/routes/trial.js. The trial wizard runs unauthenticated up to the photo step, then issues a session token tied to an anonymous user row. The session token replaces the auth Bearer for trial-only endpoints.

Pre-auth (unauthenticated)

MethodPathNotes
POST/api/trial/analyze-photoFace + body detection via Python service. Returns face thumb, body crop, faceBox.
POST/api/trial/generate-preview-avatar9:16 watercolor preview from face photo (Gemini → Grok fallback). Also extracts physical traits and seeds the trait cache (hash-keyed) so create-anonymous-account doesn't re-call Gemini for the same photo.
POST/api/trial/create-anonymous-accountTurnstile + fingerprint + daily cap. INSERTs anonymous user + character row. Stamps photo_consent_at = NOW() at INSERT time (consent already taken in the wizard checkboxes). Reads cached traits if present, else extracts. Returns { sessionToken, userId, characterId, charId }. Client typically prewarms this in the background the moment the form is valid, so the Next-button click feels instant.
POST/api/trial/generate-ideas-streamStreams story-idea candidates (Claude Sonnet). Uses landmark proximity fallback when no city-named landmarks indexed (lat/lon from /api/user/location).

Authenticated (session token)

MethodPathNotes
PATCH/api/trial/update-character-detailsSyncs name/age/gender/traits/customTraits edits made AFTER the prewarmed create-anonymous-account. Does not touch server-owned fields (physical, avatars). Client fires this on Next only if the form diffed since prewarm.
GET/api/trial/check-statusReturns trialUsed (stories_generated >= story_quota). Forced false on staging via STAGING_AUTH_PASSWORD env signal so the same trial user can re-run.
POST/api/trial/prepare-titlePre-generates 2×4 styled avatars (standard + costumed) under a runInCacheScope; the story job re-uses them. Pre-warming this here lets the story job skip Phase-2 avatar styling.
POST/api/trial/create-jobAtomic check-and-increment of stories_generated (cap dropped on staging). Schedules the unified story-generation pipeline.
GET/api/trial/job-status/:idPolled by the trial generation page; carries progress %, title page image, page images as they arrive.
POST/api/trial/link-emailTrial → real email. Sets email + verification token, sends verification mail. Stays anonymous until verified.
POST/api/trial/link-googleTrial → Google account in one step. Sets is_trial=false, anonymous=false, email_verified=true; grants welcome credits. Fires the deferred sendTrialCompletionEmailIfDeferred (PDF email skipped during pipeline because anon_*@anonymous rejects).

The unified pipeline's story-complete email send is gated by the user's email format: when it matches /^anon_.+@anonymous$/i the PDF generation and send are skipped — the deferred helper at server/lib/trialEmail.js fires it on the first claim path that produces a real verified email (verify-email handler, link-google handler). Idempotent via users.trial_completion_email_sent_at.

Trial → password conversion (POST /api/auth/set-password on a trial user) refuses with { code: 'EMAIL_REQUIRED_FIRST' } if the email is still the anon placeholder, so the resulting "full account" can't be locked-out unloginable.

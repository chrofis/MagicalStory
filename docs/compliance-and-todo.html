3. GDPR & data retention

Currently soft-delete is the only delete path. A real hard-delete needs to:

  • Wipe Postgres rows (cascade through stories, characters, story_jobs, story_images, orders, referral_events, credit_transactions, activity_log).
  • Wipe R2 objects (avatars, page images, VB references, original photos).
  • Honour Stripe / Gelato retention requirements (we can't delete invoices; need to store anonymised stubs).
  • Provide a confirmation flow + email receipt of deletion.

Implementation note: a POST /api/user/delete-account endpoint is the right shape; queue the deletion as a background job with a 7-day grace period.

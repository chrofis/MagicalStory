Text Overlay (Calm-Zone) Pipeline

Story text is overlaid directly on the page image to match the printed book layout. The pipeline picks a calm region of each image, lightens it, and renders text there.

1. Overview

  1. Scene expansion (Claude) picks a textPosition per page (one of six positions).
  2. Image prompt includes a COPY SPACE instruction telling the model to keep the chosen area calm and light.
  3. Post-generation, server/lib/textRegion.js computes a calmness heatmap from the actual image, builds a pixel-level alpha mask within the correct spread side, and composites a white wash directly into the stored image.
  4. enforceSpreadTextPosition corrects Claude's mistakes (flips left ↔ right when on the wrong side).
  5. Frontend / PDF render text on top of the (already washed) image at the detected rect.

2. textPosition decision (Claude)

One of six positions, set during scene expansion (prompts/scene-expansion.txt):

top-left      top-center      top-right
bottom-left   bottom-center   bottom-right

Claude chooses based on the scene composition. The accompanying scene prose is written so the chosen area is calm and light — e.g. an open sky for top positions, a smooth wall behind a character for bottom-side positions.

3. Spread rule (left / right by parity)

To match a printed book spread:

  • Odd page numbers (1, 3, 5, …) → text on the left side.
  • Even page numbers (2, 4, 6, …) → text on the right side.

enforceSpreadTextPosition in server/lib/storyHelpers.js flips the position if Claude picked the wrong side (e.g. bottom-right on an odd page becomes bottom-left). The COPY SPACE instruction in the image prompt then asks the model to keep that side calm.

4. Calmness detection & white wash

server/lib/textRegion.js:

  1. Loads the generated page image.
  2. Computes per-block brightness and variance — low variance + high brightness = calm + light.
  3. Builds a pixel-level alpha mask within a candidate rect on the correct spread side.
  4. Composites a white wash at 45% opacity through the mask, lightening the region without erasing the underlying art.
  5. Returns the detected rect (textRect) so the PDF renderer knows where to place text.
The wash modifies the stored image. There is no "original without wash" — this is intentional, the wash is part of the final image so frontend and PDF only need one source.

5. Frontend overlay & toggle

Two views render text on top of the washed image:

  • StoryDisplay (editor) — text overlay on by default, eye-icon toggle button to hide.
  • SharedStoryViewer — text overlay on by default with toggle, uses textPosition from the API response.

Positioning logic: client/src/utils/textOverlay.ts — six standard positions plus an explicit textRect override when the calmness detector found a non-corner zone.

6. PDF rendering

server/lib/pdf.js reads textPosition + textRect from each page and places text at the detected rect. Falls back to the corner positions if no rect is present (older stories). Same pipeline serves on-screen download and the print path.

7. Feature flags

The text-overlay pipeline is always on for new stories; only the rendering toggle is user-facing. Older stories without textRect data still render via corner fallback.

For developer testing: scripts/test-scene.js can run a single page through the current pipeline including the wash, with overlays and masks rendered into the output directory.

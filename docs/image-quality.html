1. Bounding-box detection

Bbox detection is the foundation: it tells every downstream check which characters and props are where. Implementation: server/lib/entityConsistency.js + Python cascade detector.

Two-pass flow

  1. Pass 1 — Gemini 2.5 Flash runs the prompt in prompts/bbox-detection.txt and returns figures with bbox: [ymin, xmin, ymax, xmax] normalised 0-1.
  2. Pass 2 (refine) — Cascade face detector (Python anime + Haar) runs on the same image; results merged with Gemini bboxes. Catches faces Gemini missed.
  3. Object name canonicalisation — detected names mapped back to Visual Bible IDs (e.g. "the wooden bow" → ART012).

Persistence

Bbox results land on sceneImage.bboxDetection per page. Not shared across versions: each round of repair runs fresh detection on the new image. Reusing sharedBboxDetection across rounds caused a long-running bug where v3's bbox showed v0's figures.

2. Three evaluators

Run in parallel after image generation:

EntityGemini + Python cascadefinalChecksReport.entity with severitiessee §1

Evaluators are tolerant by design: pose mirror (left/right body-part swap), minor framing nits, and aesthetic preferences don't deduct. The decision log records which deductions were rejected and why.

3. The unified score

One number drives every redo decision:

finalScore = qualityScore − semanticPenalties − entityPenalties

Severity bands (used by both eval paths and the consolidator):

SeverityDeduction
CRITICAL−30
MAJOR−15 (entity: −20)
MODERATE−7
MINOR−2 (entity: −10)

Quality-eval issues are NOT double-counted — they're already inside qualityScore. The dev panel reconciles by showing Endwert = max(0, 100 − totalPenalty) alongside the raw backend score; if they differ, the backend score is shown as a small annotation.

Final score from the consolidator

server/lib/feedbackConsolidator.js validates and clamps plan.final_score to [0, 100] and stores final_score_reason. findBadPages in server/lib/repairLogic.js prefers consolidatorFinalScore when present.

4. Repair workflow loop

Triggered by RepairWorkflowPanel → useRepairWorkflow.runFullWorkflow(). Up to maxPasses = 3 global passes:

Pass 1:
  EVALUATE ALL pages          (quality + semantic + entity-from-generation)
  IDENTIFY BAD PAGES          (score < 60 OR issues >= 5)
  REDO bad pages once each    (POST /:id/iterate/:pageNum)

Pass 2..3:
  RUN ENTITY CONSISTENCY      (fresh bbox on new images)
  EVALUATE ALL pages          (with fresh entity data)
  IDENTIFY BAD PAGES
  REDO bad pages once each

FINAL:
  ENTITY CONSISTENCY (last)
  EVALUATE ALL (final)
  CHARACTER REPAIR (max 3 pages, critical first, Grok blended)
  PICK BEST VERSIONS

Thresholds in server/config/models.js → REPAIR_DEFAULTS:

scoreThreshold60 — pages below this get redone
issueThreshold5 — pages with 5+ fixable issues get redone
maxPasses3 — global passes
maxCharRepairPages20 — max pages for char repair per run
The threshold dropped from 80 → 60 to stop redoing pages with cumulative-but-minor issues. Three checks each deducting 5 points used to trigger a costly redo for what was actually a fine image.

5. Character repair

When entity consistency flags a character mismatch (wrong hair, wrong outfit, wrong age), character repair fixes only that character, preserving the background. Three modes (server/lib/images.js → repairCharacterMismatchWithGrok):

ModeHowCostWhen
Cutout (default)Extract bbox + 20% padding → send cutout + avatar to Grok → composite back with feathered edge (~8% of smaller dimension).$0.02Background untouched. Default for body / outfit fixes.
BlendedBlur character region → Grok regenerates → 30px feathered blend onto bbox + 50% padding.$0.02Face-only repairs.
BlackoutFull scene with blackout overlay → Grok regenerates entire scene.$0.02Background and character both wrong.

Preset-aligned cutout

computePresetAlignedExtract() picks one of Grok's 13 aspect presets and grows one axis to match — no letterbox padding, just more scene context.

6. Scale repair

Separate from character repair. Used when a character is correctly rendered but in the wrong place / wrong size (e.g. should be a small figure in the background, drawn as foreground). server/lib/scaleRepair.js.

Prompt is framed as MOVE / RELOCATE, not ADD: "every named character is already drawn somewhere in the input image — do not add new figures; find the existing figure of [visual identifier] and MOVE+SHRINK it to the target position [metadata position]". Repaints the abandoned area. No name leakage (uses visual identifiers, not character names).

7. Pick-best version selection

After all repair passes finish, every page has 1..N versions stored under imageVersions[]. POST /:id/repair-workflow/pick-best-versions activates the highest-scoring version per page. Character-repair outputs are also versions, so they compete with the original + redo attempts.

8. Decision log (kept & rejected knobs)

Curated decisions for grok_inpaint and the wider repair stack. Each entry: ✅ kept / ❌ rejected / 🟡 conditional + one-line reason.

KnobVerdictReason
Grok inpaint for body / pose / gaze repairs✅Handles structural changes (body rotation, pose, gaze) when instructions use visual identifiers, not names.
Pose mirror penalty in semantic eval❌Left/right body-part swaps (foot, arm, hand) are not semantic defects. Strengthen the prompt instead.
Position phrasing as "left foreground / right midground"❌Use relational phrasing ("on the boat", "behind Roger") unless cross-page continuity / text-zone / ultra-wide demands LR.
"For white text" in prompts❌Says "calm/smooth" instead. Saturated colour fine; don't bake dark-band language into unified prose.
Single bbox detection reused across rounds❌Caused v3 to inherit v0's bbox. Each round runs fresh detection.
scoreThreshold = 80❌Redid too many pages with merely cumulative minor issues. Lowered to 60.
generateEmptyScenes = true❌Empty plates didn't help final quality enough to justify the call. Default off.
Hatch / mask shape variations on cutout❌Tested several mask shapes; feathered rectangle outperformed.
Bbox + 40% min padding for cutout✅Smaller padding clipped hands and hair; larger pulled in too much background.
Per-page version accumulation✅Don't overwrite — let pick-best compare all versions.

You are evaluating a 2×4 character reference sheet produced by an image model.

You receive UP TO THREE images:
  - Image 1: the SOURCE FACE PHOTO of the real person the sheet should depict.
  - Image 2: the STANDARD AVATAR — a single-image body reference of the SAME PERSON in modern standard clothing. Use it as an identity anchor alongside Image 1 (body proportions, apparent age, build, identity confirmation). Only present when supplied — when absent, fall back to using Image 1 + the character profile as the only anchors.
  - Image LAST: the GENERATED 2×4 sheet to evaluate. When all three images are supplied, this is Image 3; when no avatar is supplied, it's Image 2.

CHARACTER_PROFILE_BLOCK

Image LAST is a single grid: 4 columns × 2 rows = 8 cells, separated by thin white gutters. Intended layout:

  Row 1 (cells 1–4, top):     HEAD AND NECK ONLY. No shoulders, no torso, no arms, no clothing. Four facing angles: front, three-quarter, profile, back of head.
  Row 2 (cells 5–8, bottom):  FULL BODY head to feet, wearing the requested costume. Same four facing angles.

A common failure: the model draws 4 full bodies in a single row and the horizontal gutter cuts each figure across the waist. That is INVALID — there must be 4 head portraits in the top row, separate from 4 full bodies in the bottom row.

Your job: rate the sheet on FOUR checks and return a JSON verdict.

TASK 1: LAYOUT
- Top row: 4 head-and-neck portraits, no shoulders or torso. Score 1-10.
- Bottom row: 4 full-body figures, head to feet. Score 1-10.
- Horizontal gutter between rows: clean white space, no figure crosses it. Score 1-10.
- Cell containment: every figure inside its own cell with white margin on all four sides. No hair, hand, foot, or shadow extends across a cell border. A figure whose head is in the top row and feet are in the bottom row is the most common failure — score 1-2 if you see this in any cell. Score 1-10.
- `layoutScore` = lowest of the four. A waist-cutting failure or any figure spanning two cells must score 1-3.

TASK 2: IDENTITY MATCHES THE STANDARD AVATAR (per cell, lowest wins)
WHEN Image 2 (standard avatar) IS PROVIDED:
- Image 2 is the IDENTITY ANCHOR — a known-good rendering of the same person. Compare EVERY cell of the sheet against Image 2, not against cell 1 of the sheet itself.
- For each of the 8 cells in the sheet (Image LAST): does it show the same person as Image 2? Same face structure, hair colour and length, skin tone, gender, age bucket.
- Pose / framing / render-angle differs by design (front, three-quarter, profile, back, head-only, full-body) — do NOT penalise that. Score by IDENTITY only.
- Per-cell score 1-10:
   * 8-10: clearly the same person; only pose/angle differs.
   * 4-6: ambiguous — face structure similar but age, hair, or build visibly drifted from Image 2.
   * 1-3: clearly a different person from Image 2 (wrong gender, wrong race, or face structure unrelated).
- Same person rendered slightly younger or older than Image 2 is NOT a 1-3; it's a 4-7 depending on degree.

WHEN no standard avatar is provided:
- Pick cell 1 of the sheet as the reference.
- For each of cells 2-8: same face structure, hair, gender, skin tone as cell 1. Pose may differ.
- Per-cell score 1-10 with the same buckets above.

`identityScore` = LOWEST of all 8 (or 7 when falling back) per-cell scores. Do not average. Do not round up.

TASK 3: OUTFIT MATCHES REQUESTED COSTUME (item-by-item)
- Read REQUESTED_OUTFIT below and split it into individual items.
- For each item, check whether the figures in cells 5-8 visibly wear it.
- All items present = 10. ONE listed item missing = ≤4. TWO+ missing = ≤2. Wrong colour on any item = ≤5.
- Cross-cell consistency: all 4 body cells in 5-8 wear the same outfit. If outfit varies between cells, score 1-3.
- `outfitScore` = LOWER of item-match and cross-cell consistency.

TASK 4: SOURCE PHOTO IDENTITY MATCH (per head cell, lowest wins)
This task is SEPARATE from Task 2. Task 2 checks "does the sheet match the standard avatar"; this task checks "does the rendered character match the real person in Image 1 + the declared profile".
- Compare Image 1 (source face photo) against EACH of the 4 head cells (cells 1, 2, 3, 4).
- When a CHARACTER PROFILE block was supplied above, also cross-check apparent age in the rendered cells against the profile's age bucket. If the profile says young-teen (12-15) and the head cell looks 6-9, score that cell 1-3 regardless of how good the face features look.
- Per-cell score 1-10. Different person (wrong gender, wrong age decade, wrong race, unrelated face) must score 1-3.
- Style/medium difference (photo vs illustration) scores 8-10 ONLY when face structure + hair + age bucket + gender ALL match. Drop to 6-7 if any one is visibly off. Drop to ≤5 if 2+ are off.
- `sourceMatchScore` = LOWEST of the 4 per-cell scores. Do not average.

TASK 5: CLEAN RENDER (no stray markings)
- Scan every cell for marks on the person that the outfit did not request: arrows, symbols, letters, numbers, logos, badges, motifs, decals, or coloured graphics on skin, hair, face, or clothing. The back-of-head and back-body cells are the most common place for a stray motif.
- `cleanScore` 1-10: 9-10 = none. 4-6 = a faint or small stray mark on one cell. 1-3 = a clear graphic/symbol on any cell (e.g. a coloured shape on the back of the head).

`finalScore` = LOWEST of layoutScore, identityScore, outfitScore, sourceMatchScore, cleanScore.
`valid` = (finalScore ≥ 6).

You are evaluating a 2×4 character reference sheet produced by an image model.

You receive TWO images:
  - Image 1: the SOURCE face photo of the real person the sheet is supposed to depict.
  - Image 2: the GENERATED 2×4 sheet to evaluate.

Image 2 is a single grid: 4 columns × 2 rows = 8 cells, separated by thin white gutters. The intended layout is:

  Row 1 (cells 1–4, top):     HEAD AND NECK ONLY — no shoulders, no torso, no arms, no legs, no clothing visible below the collarbone. Four facing angles: front, three-quarter, profile, back of head.
  Row 2 (cells 5–8, bottom):  FULL BODY — head to feet, wearing the requested costume. Four facing angles in the same order.

A common failure: the model draws 4 full bodies in a single row and the horizontal gutter cuts each figure across the waist. That is INVALID — there must be 4 head portraits in the top row, separate from 4 full bodies in the bottom row.

Your job: rate the sheet on FOUR checks and return a JSON verdict. If ANY check fails, the sheet is rejected and regenerated.

TASK 1: LAYOUT
- Top row contains 4 head-and-neck portraits only. No shoulders or torso visible. Score 1-10.
- Bottom row contains 4 full-body figures, head to feet. Score 1-10.
- The horizontal gutter between rows runs through clean white space — no figure crosses it. Score 1-10.
- Cell containment: every figure stays fully inside its own cell with white margin on all four sides. No hair, hand, foot, or shadow extends across a cell border. A figure whose head is in the top row and feet are in the bottom row is the most common failure — score 1-2 if you see this in any cell. Score 1-10.
- `layoutScore` = lowest of the four. A waist-cutting failure or any figure spanning two cells must score 1-3.

TASK 2: IDENTITY CONSISTENCY (per cell, lowest wins)
- Pick the person depicted in the front cell (cell 1) as the reference. Score every OTHER cell against it individually.
- For each of cells 2, 3, 4 (top-row head cells): does it show the same person as cell 1? Same face shape, eyes, nose, mouth, skin tone, hair colour, hairstyle, glasses (if any), AND same apparent gender. Only the camera angle should differ. Score each cell 1-10.
- For each of cells 5, 6, 7, 8 (bottom-row body cells): does it show the same person as the heads (same face when visible, same gender, same age, same proportions, same skin tone)? Score each cell 1-10.
- `identityScore` MUST equal the LOWEST of all 7 per-cell scores. ONE cell showing a different person (wrong gender, different face, different age) must score 1-3 for that cell and therefore for identityScore — even if the other 6 cells are perfect. Do not average. Do not round up.

TASK 3: OUTFIT CONSISTENCY (only if REQUESTED_OUTFIT is given below)
- All 4 full-body figures wear the SAME outfit — every accessory, every garment colour, every cut. Score 1-10.
- The outfit matches REQUESTED_OUTFIT (when provided): right type, right colours, right pieces. Score 1-10.
- `outfitScore` = lower of the two. Different outfits across cells must score 1-3.
- If REQUESTED_OUTFIT is not provided, set `outfitScore` based on cross-cell outfit consistency alone (no match check).

TASK 4: SOURCE-PHOTO IDENTITY MATCH (per head cell, lowest wins)
- Compare Image 1 (source photo) against EACH of the 4 head cells in Image 2 (cells 1, 2, 3, 4) individually.
- For each head cell: is this the SAME PERSON as the source photo? Same face structure (jawline, brow, cheekbones), same eye spacing/colour, same nose shape, same hair colour & general style, same skin tone, same apparent age range, same gender. Style/medium can differ (photo vs illustration) — what matters is recognisable identity.
- Score each cell 1-10. Different person (wrong gender, wrong age decade, wrong race, completely different face) must score 1-3 for that cell.
- `sourceMatchScore` MUST equal the LOWEST of the 4 per-cell scores. ONE wrong-person cell out of four must drag sourceMatchScore down to that cell's score. Do not average. Do not round up.
- Same person rendered in a different style = 8-10 per cell. Identity "kinda there" but several wrong details in a cell = 4-6 for that cell.

`finalScore` MUST equal the LOWEST of layoutScore, identityScore, outfitScore, sourceMatchScore.
`valid` is `true` only when finalScore ≥ 6.

COORDINATE SYSTEM:
- Values are 0-1000 (Gemini native format)
- Origin is top-left corner
- Format: [y_min, x_min, y_max, x_max]

FIGURE COUNT LIMIT:
- Return AT MOST 10 figures total. If the scene has a crowd in the background, do NOT enumerate each crowd member as a separate figure — the crowd is backdrop, not a list.
- Only include a figure if its face OR body occupies at least ~5% of the frame width or height. Tiny silhouettes in the distance are backdrop, skip them.
- If you cannot distinguish between several figures (e.g. multiple blurred robes in the background), return ONE figure representing the group and set `name: "UNKNOWN"`, `label: "background crowd"` — never duplicate the same label across many entries.

FOR EACH FIGURE (up to 10, including partially visible ones near the edges of main subjects):
- If you can identify it as an expected character, set "name" to the character name (EXACTLY as listed above)
- Use CLOTHING + POSITION to distinguish figures (two figures with similar build are distinguished by what they wear and where they stand)
- If not identifiable as any expected character, set "name" to "UNKNOWN"
- Include "confidence": "high", "medium", or "low" for identification certainty
- face_box: bounding box around the head region.
  Include the top of the head and the bottom of the head outline, and the full width from side to side.
  Common mistakes to AVOID:
  * Do NOT cut off the top of the head — include the full head outline
  * Do NOT cut off the bottom of the head outline
  * Do NOT make the box too narrow — include the full head width
  * When in doubt, make the face_box LARGER rather than smaller
  * The face_box should be roughly square for a front-facing figure
- body_box: bounding box around the FULL FIGURE (or as much as visible)
- If the head is not visible (back view, non-human creature), set face_box to null

FOR EXPECTED OBJECTS:
- Report "found": true/false for each expected object
- If found, include body_box bounding box and position

LABEL LENGTH RULE (non-negotiable):
- Every `label` field MUST be ≤ 10 words.
- Name ONE defining clothing item + ONE distinguishing feature. Example: "boy in blue hoodie with backpack".
- DO NOT list everything the figure is wearing. DO NOT describe actions. DO NOT describe expressions.
- Long labels waste response tokens and cause the JSON to truncate before closing.

Return ONLY valid JSON:

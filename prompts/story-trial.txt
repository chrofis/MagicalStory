# Scene Hint Format

Each scene hint is a JSON block for direct image generation (no further expansion).
It must be detailed enough to illustrate without additional context.

```json
{
  "scene": {
    "imageSummary": "[2-3 sentences: the visual moment with character positions, mood, colors, lighting, atmosphere]",
    "setting": {
      "location": "[Location Name [LOC001]]" or "none",
      "indoorOutdoor": "[indoor/outdoor]",
      "description": "[visual setting details: colors, textures, atmosphere]",
      "lighting": "[time of day + light quality]",
      "weather": "[sunny/cloudy/rainy/snowy/n/a]",
      "camera": "[close-up|medium|wide|ultra-wide]",
      "depthLayers": "[describe what's in foreground, midground, background]"
    },
    "characters": [
      {
        "id": "[CHR001 or null for main character]",
        "name": "[Name]",
        "position": "[left/right/center]",
        "action": "[what they're doing]",
        "expression": "[facial expression]",
        "clothing": "[standard | costumed]",
        "depth": "[foreground|midground|background]",
        "perspective": "[front|side|back view|over-the-shoulder]"
      }
    ],
    "objects": [
      {"id": "[OBJ001]", "name": "[Object name]", "position": "[where in scene]"}
    ],
    "background": "[empty|a few passersby|busy crowd|dense forest|etc.]"
  }
}
```

- MAX 4 characters per scene. Use character names, not descriptions.
- `characters.id`: use visual bible `CHR` IDs for secondary characters (e.g., `"CHR001"`). Use `null` for the main character (who has reference photos).
- `characters.depth` (optional, default `foreground`): how far from camera. Use `background` for tiny/distant figures.
- `characters.perspective` (optional, default `front`): camera angle on this character.
- `setting.location`: use `"Name [LOC001]"` for the default photo. When the landmark in the LANDMARKS section lists PHOTO ANGLES, use `"Name [LOC001.N]"` where N is the variant whose description fits the scene framing (interior shots for "on/in the landmark" scenes, exterior shots for distant/establishing views).
- `setting.camera`: shot type determines facial detail visibility. `close-up`/`medium` = expressions visible. `wide`/`ultra-wide` = body language only.
- `objects`: include visual bible items visible in the scene with their IDs.
- `background`: describes crowd/environment density. Tells image generator if scene feels isolated or populated.
- Use `"n/a"` for weather in indoor scenes.

Don't include the main character ({MAIN_CHARACTER_NAME}) — they have reference photos.
Only include characters that you invent (friends, shopkeepers, etc.) as `secondaryCharacters`.
Use empty arrays `[]` for categories with no elements.

**Real-world landmarks:** Any location whose name comes from (or is recognisably) an entry in the LANDMARKS section above is a real-world landmark. For every such location:
- `"isRealLandmark": true` (not false — if you wrote the landmark's name, you picked it from the list, so it is real)
- `"landmarkQuery"`: the exact landmark name from the LANDMARKS section. Copy it verbatim — do not paraphrase, do not drop parens, do not anglicise.

This is the difference between the system fetching a real Wikipedia photo of the actual building (correct) and the model inventing a generic illustration of what it thinks the building looks like (wrong). When in doubt, mark it as real — false negatives waste the photo pipeline and produce off-brand renders.

Example — picking "Kapellbrücke (Luzern) [Bridge]" from the LANDMARKS section:
```json
{
  "id": "LOC001",
  "name": "Kapellbrücke",
  "pages": [1, 2, 3],
  "setting": "outdoor bridge over the Reuss river",
  "isRealLandmark": true,
  "landmarkQuery": "Kapellbrücke (Luzern)"
}
```
Note: `name` can be styled for your story, but `landmarkQuery` must match the LANDMARKS entry character-for-character (parentheses and umlauts included).

**BACKGROUNDS (style anchors for each scene):**
Each background describes a scene with no people, no human figures. Include: location details, architecture, objects, lighting, weather, camera angle. Leave open space where characters would stand. Backgrounds can be reused across multiple pages. Every page must have a background.
- Use the landmark photo descriptions if available to describe real locations accurately
- For indoor scenes: describe furniture, walls, lighting, floor
- For outdoor scenes: describe landscape, sky, vegetation, architecture

**Keep it MINIMAL for a 5-scene story:**
- max **2 locations**
- max **2 artifacts** (only props that actually drive the action across multiple scenes — not background dressing)
- max **2 secondary characters** (only if they appear on 2+ pages — single-scene helpers can stay nameless in prose)
- 3-5 backgrounds (one per scene if scenes change settings, fewer if scenes share a setting)

Every entry costs a reference image generation pass. A 5-page story rarely benefits from more than 2 of each — pick the elements that need consistency across scenes and leave the rest as scene-prose details.

# Google Ads CLI — Setup

Goal: script-driven control over the MagicalStory Google Ads account, bypassing the "recommended campaign" UX in the web UI.

This folder starts minimal — just the connection. Campaign/creative sync comes later.

---

## One-time setup (do this once, takes 30 minutes plus 1-3 days wait on the dev token)

You need **four credentials**:

| | What | Where to get it | Time |
|-|------|-----------------|------|
| 1 | **Google Ads Customer ID** | ads.google.com top-right, 10 digits like `123-456-7890` | 1 min |
| 2 | **Developer Token** | ads.google.com → Tools & Settings (wrench icon) → Setup → API Center → Apply | 1-3 days approval |
| 3 | **OAuth Client ID + Client Secret** | console.cloud.google.com → APIs & Services → Credentials → Create OAuth 2.0 Client → Application type: **Desktop app** | 5 min |
| 4 | **Refresh Token** | Generated by running `node scripts/ads/authorize.js` after steps 1-3 | 2 min |

### Step 1 — Find your Customer ID
Log into ads.google.com. Top-right corner shows `123-456-7890`. Drop the dashes → that's your customer ID.

### Step 2 — Apply for a Developer Token
1. ads.google.com → wrench icon (Tools & Settings) → **Setup** → **API Center**
2. Fill the form. Use case: "Internal tooling for managing our own Google Ads account." Company: your legal name.
3. You get a **Basic Access** token immediately (works with test accounts only) and then **Standard Access** after approval (1-3 business days).
4. Token looks like `AbCdEf-1234567890_GhIjKl`.

### Step 3 — Create OAuth credentials
1. console.cloud.google.com → **APIs & Services** → **Credentials**
2. **+ CREATE CREDENTIALS** → **OAuth client ID**
3. If prompted, configure an OAuth consent screen first: User type = **External**, app name = "MagicalStory Ads CLI", user support email = your email, scopes = leave default, test users = add your Google Ads account email.
4. Application type: **Desktop app**. Name: "MagicalStory Ads CLI".
5. Download the JSON or copy the `client_id` and `client_secret`.

### Step 4 — Configure and authorize
```bash
cp scripts/ads/config.example.json scripts/ads/config.json
# edit scripts/ads/config.json with customer ID, dev token, client ID, client secret
# leave refresh_token empty for now
node scripts/ads/authorize.js
# opens a browser, you log in and consent, it prints a refresh token
# paste the refresh token into config.json
```

---

## Test the connection
```bash
node scripts/ads/whoami.js
```
Should print your account name, currency, timezone, and a count of accessible customers. If this works the connection is done — everything else (campaign creation, creative sync, geo targeting) is a layer on top.

---

## Files in this folder

- `config.example.json` — template (safe to commit).
- `config.json` — your secrets (git-ignored, never commit).
- `authorize.js` — one-time OAuth flow to mint the refresh token.
- `whoami.js` — sanity check: hits the API, prints account info.
- `lib/client.js` — thin wrapper that reads config.json and returns a ready client.
